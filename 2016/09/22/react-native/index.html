组件 Component

生命周期

  • state: 组件的属性 主要是用来存储组件自身所需要的数据 每次组件的更新都是通过修改组件state的值 ReactJS内部会监听组件state的变化，一旦发生变化，则会主动出发render方法来更新虚拟DOM结构

  • 虚拟DOM: 将真实的DOM结构映射成一个JSON数据结构

创建阶段

getDefaultProps: 在创建类时被调用 主要用于处理组建的默认值（this.props）
在React.createClass调用。

实例化阶段

React.render() 后开始实例化阶段

  • getInitialState: 初始化组建的state值，其返回值赋值给组建的state（获取组建的state默认值）

  • componentWillMount: 在render之前调用此方法 业务逻辑处理应该放在这里（譬如对state的操作）

  • render: 根据state的值，渲染并返回一个虚拟DOM

  • componentDidMount: 该方法在render之后 reactJS会使用render方法返回的虚拟DOM对象来创建真实的DOM结构
    组建内部可以通过this.getDOMNode()来获取当前组件的节点

更新阶段

  • componentWillRecieveProps: 该方法发生在this.props被修改 或者父组件调用setProps()方法之后 或调用this.setState对state进行修改

  • shouldComponentUpdate: 该方法用来拦截新的prop或者state，根据逻辑来判断是否需要更新

  • componentWillUpdate: shouldComponentUpdate返回true时执行，组件将更新

  • componentDidUpdate: 组件完成更新，可以在这里进行DOM操作

销毁阶段

  • componentWillUnmount: 销毁时进行调用，取消事件绑定，移除虚拟DOM对应的数据结构、销毁一些无效的定时器

另外，React 组件中最重要的一个概念就是 state – 组件的状态。除了前面的使用 getInitialState 方法来设置初始化状态外。在界面逻辑处理或者事件交互的过程中，可以调用 this.setState(…) 方法来修改组件的状态值。
